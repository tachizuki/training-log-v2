# 身体写真・ポージング AI 分析機能 設計メモ

優先度: 低（コスト要相談）。本ドキュメントは将来の実装に向けた設計案を残すためのもの。
現時点では UI/API ともに未実装。

## 1. ユースケース

- 大会前の仕上がり確認（フロント／サイド／バック）
- 経時的な体型比較（同じポーズの月次比較）
- ポージング練習の改善点抽出（規定／フリーポーズ）

## 2. 入力フォーマット

### 2.1 静止画

- フロント／サイド／バックの 3 枚を想定
- 解像度は長辺 1280px までリサイズしてからアップロード
- フォーマット: JPEG / WebP
- ストレージ: 端末ローカル + Firebase Storage（プレミアム限定機能）

### 2.2 動画（ポージング分析）

- 規定ポーズ通しを 30 秒〜2 分程度の動画で
- フォーマット: MP4 (H.264) / 解像度 720p
- 端末側で 1fps 程度にサンプリングしてからフレームを送るとコスト削減になる

**画像 vs 動画の判断**: ポージング分析は「動き」と「ホールド時間」が重要なため、動画推奨。
ただしコスト・通信量が大きいので、まずは「ポーズ切替の頂点」をユーザーがタップで切り出した
静止画 3〜5 枚で代替する MVP を検討する。

## 3. 想定 API

- Claude / Gemini / GPT-4o などの Vision API のいずれか
- 出力は構造化 JSON で受け取る

例:

```json
{
  "overall_score": 78,
  "body_parts": {
    "shoulders": { "score": 82, "note": "三角筋の張りは良いが前部がやや弱い" },
    "back":      { "score": 70, "note": "ラットの広がりは良いがディテール不足" },
    "legs":      { "score": 65, "note": "前から見ると外側スイープがもう一歩" }
  },
  "balance": "上半身が下半身より発達して見える",
  "posing": {
    "fc_pose":   { "score": 75, "note": "肘の引きが甘い" },
    "side_chest":{ "score": 80, "note": "クランチが浅い" }

  "advice": [
    "下半身を強化（特にスイープ）",

  ]

```

## 4. プライバシー設計

- 写真・動画は端末側に暗号化保存
- クラウドアップロードはオプトイン（プロフィールで明示同意）
- 分析後のアップロードファイルは N 日（例: 7 日）で自動削除
- 分析結果（テキストとスコアのみ）は長期保存

## 5. コスト見積もり（ざっくり）

- 静止画 3 枚 + 1024px → 1 回約 ¥10〜30 / 月（1〜2 回想定）
- 動画 30 秒 1fps 30 枚 → 1 回 ¥100〜300
- 月 1 回想定 → 月額プレミアムプラン（¥300/月）に内包可能

## 6. UI ラフ案

```
プロフィール画面
└ 📸 身体スキャン (Premium)
   ├ 直近のスキャン: 2026/05/14（写真 3 枚、スコア 78）
   ├ ＋ 新しいスキャン
   │   └ フロント／サイド／バックを撮影 → 1024px に圧縮 → AI 解析
   └ 比較ビュー（前回 vs 今回）

筋トレ画面
└ 📹 ポージング分析 (Premium)
   ├ 動画を撮影 / アップロード
   ├ ポーズ切替の頂点を 3〜5 枚タップで切り出し
   └ AI 解析 → ポーズ別スコア & アドバイス
```

## 7. 実装フェーズ

| フェーズ | 内容 | 優先度 |
| --- | --- | --- |
| Phase 1 | 写真の端末ローカル保存と一覧表示のみ（AI なし） | 中 |
| Phase 2 | AI 静止画分析（フロント / サイド / バック） | 低 |
| Phase 3 | 動画ポージング分析（あるいはタップ切り出し方式） | 低 |
| Phase 4 | 経時比較ビュー（過去 N 回との重ね表示） | 低 |

実装には課金・プライバシー設計の整理が先行して必要。
