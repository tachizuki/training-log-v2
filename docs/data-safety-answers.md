# Google Play データセーフティ 回答シート（PhysiqueLog）

Play Console →「アプリのコンテンツ」→「データセーフティ」で、以下の通り回答する。
（前提：広告SDK・第三者解析なし。同期はFirebase/Firestore。決済はGoogle Play。）

---

## 1. データ収集と共有（概要）
- **このアプリはユーザーデータを収集または共有しますか？** → **はい（収集する）**
- **第三者と共有しますか？** → **いいえ**
  - ※ Firebase/Firestoreは「自社サービスの提供のための処理（インフラ）」であり“共有”には該当しない。決済はGoogle Playが処理。お問い合わせはGoogle Apps Script（自分の受信箱）に送るのみ。広告・解析目的の共有は無し。

## 2. 収集するデータ種別（「収集」にチェックする項目）

| カテゴリ | データ種別 | 収集 | 用途 | 任意/必須 |
|---|---|---|---|---|
| 個人情報 | **メールアドレス** | ✅ | アカウント管理（ログイン） | 任意（ログイン時のみ） |
| 個人情報 | 名前（表示名） | ✅ | アカウント管理 | 任意（ログイン時のみ） |
| 健康とフィットネス | **健康情報 / フィットネス情報**（体重・食事・水分・睡眠・歩数・トレ記録） | ✅ | アプリ機能（記録・分析・同期） | 任意 |
| アプリのアクティビティ | アプリ内の操作・設定 | ✅（任意） | アプリ機能 | 任意 |
| アプリ情報とパフォーマンス | 診断情報（お問い合わせ時のバージョン等） | ✅（任意） | 不具合対応 | 任意 |

- ※「金融情報（クレジットカード等）」は**収集しない**（Google Playが決済処理）。
- ※「位置情報」「連絡先」「写真/動画」「メッセージ」等は**収集しない**。

## 3. 各データ種別の詳細（Consoleで各項目に答える内容）

共通の回答：
- **このデータは収集（端末外へ送信）されますか？**
  - ログイン＋クラウド同期を使う場合 → **はい（収集）**。未ログインなら端末内のみ＝送信されない。Consoleでは「収集する」を選び、下記を回答。
- **このデータは暗号化されて送信されますか？** → **はい**（HTTPS）
- **ユーザーはデータの削除をリクエストできますか？** → **はい**（アプリ内「アカウント削除」で全削除）
- **このデータ収集は必須ですか？** → **いいえ（任意）**（ログイン/同期は任意機能）

データ種別ごとの「用途（purpose）」：
- メールアドレス / 名前 → **アカウント管理**
- 健康・フィットネス情報 → **アプリの機能**（必要なら「アプリの機能」のみでOK。分析機能は自分の端末/アカウント内処理）
- アプリのアクティビティ / 診断 → **アプリの機能** /(診断は)**不具合対応**

## 4. セキュリティ慣行（Consoleの該当質問）
- **転送中のデータは暗号化されますか？** → **はい**
- **ユーザーはデータ削除をリクエストできますか？** → **はい**（アカウント削除機能）

## 6. 補足・注意
- 申告内容は**実装と一致**している必要がある（虚偽はポリシー違反）。本シートは現状の実装（Firebase auth/firestore、広告/解析なし、削除機能あり、HTTPS）に基づく。
- 将来 **Firebase Analytics / Crashlytics / 広告** 等を追加したら、データセーフティを更新すること。
- 「健康情報」を扱うため、ストアのデータセーフティ表示でも健康カテゴリが出る。プライバシーポリシーで取扱いを明記済み。
